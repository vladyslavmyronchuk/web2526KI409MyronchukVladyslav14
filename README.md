LAB5 - HTTPS

Hardware LAB5:
Мікроконтролер ESP32 DebKit v1: Використовується як обчислювальне ядро, здатне виконувати ресурсомісткі криптографічні операції шифрування та дешифрування в реальному часі.
Вбудована Flash-пам'ять: Використовується для зберігання сертифікатів безпеки X.509 та статичних ресурсів захищеного вебінтерфейсу через файлову систему LittleFS.
Wi-Fi модуль: Забезпечує бездротовий канал зв'язку для передачі зашифрованого TLS-трафіку.
Internal LED: Виступає як об'єкт захищеного керування, доступ до якого надається лише після проходження автентифікаці

ПЗ та бібліотеки LAB5:
OpenSSL (v3.x): Професійний пакет інструментів, використаний на ПК для генерації приватного ключа RSA (2048-bit) та самопідписаного сертифіката сервера (server.crt).
mbedTLS : Криптографічна бібліотека, що забезпечує реалізацію стека TLSv1.2 для шифрування трафіку за стандартом AES-256.Вебсервіси та автентифікація
ESP32_HTTPS_Server: Високорівнева бібліотека для запуску захищеного вебсервера на стандартному порті 443.
Basic Authentication: Механізм перевірки облікових даних користувача через заголовок Authorization. Доступ надається лише за наявності валідного Base64-хендшейку .
FrontendLittleFS : Файлова система для зберігання вебінтерфейсу, доступ до якого обмежений правами авторизації.
ArduinoJson (v7.2.2): Використовується для обміну даними про стан системи в зашифрованому форматі JSON.
Modern JS (Fetch API): Реалізує асинхронні запити до захищеного API контролера, підтримуючи SSL-з'єднання на стороні браузера.
