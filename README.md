LAB3 - WEBSOCKET

Hardware lab3: Мікроконтролер ESP32 DevKit v1: Виступає як центральний обчислювальний модуль, що одночасно підтримує роботу HTTP-сервера та WebSocket-сервера. Датчик магнітного поля HMC5883L: Цифровий трьохосьовий магнітометр, обраний відповідно до 4-го варіанту завдання. Інтерфейс підключення I2C:GPIO 21 (SDA) — лінія даних.GPIO 22 (SCL) — лінія синхронізації Модуль Wi-Fi: Забезпечує бездротовий зв'язок між клієнтом (браузером) та контролером для передачі HTTP-пакетів.

ПЗ та бібліотеки lab3: LittleFS : Енергонезалежна файлова система для автономного зберігання статичних ресурсів вебінтерфейсу Flash-пам'яті. WiFi.h : Забезпечує підключення до локальної мережі Wi-Fi для мережевої взаємодії. WebSocketsServer v2.x: Реалізує серверний ендпоїнт на порті 81. Забезпечує повнодуплексний зв’язок, що дозволяє контролеру самостійно транслювати дані клієнту без постійних опитувань. WebServer.h: Використовується для початкової видачі вебінтерфейсу на стандартному порті 80.ArduinoJson v7.2.2: Використовується для пакування значень осей X, Y, Z у формат JSON-пакета для передачі через WebSocket .

Frontend: QMC5883LCompass v1.2.3: Спеціалізована бібліотека для ініціалізації датчика та зчитування цілочисельних значень магнітної індукції . JavaScript (WebSockets API): Обробляє постійне з’єднання. Через обробник onmessage приймає JSON-пакети та оновлює стан інтерфейсу . Chart.js (v4.x via CDN): Використовується для рендерингу лінійного графіка . Алгоритм Rolling Window: Програмна реалізація зсуву графіка вліво шляхом видалення застарілих даних (понад 50 точок) для підтримки високої продуктивності системи .

LAB4  - CAPTIVE PORTAL

Hardware lab4:
Мікроконтролер ESP32 DevKit v1: Виступає в ролі інтелектуального мережевого вузла, що підтримує режими Access Point  для налаштування та Station  для основної роботи.
Модуль Wi-Fi: Інтегрований у контролер, забезпечує створення власної точки доступу  та підключення до зовнішніх роутерів.
Енергонезалежна пам’ять : Використовується для надійного зберігання облікових даних  навіть після вимкнення живлення пристрою.
USB-інтерфейс: Використовується для живлення та моніторингу стану пристрою через Serial-порт.


ПЗ та Бібліотеки lab4:
LittleFS : Використовується для зберігання файлу config.html, який містить вебінтерфейс Captive Portal .
Preferences.h : Бібліотека для зручної роботи з енергонезалежною пам'яттю (NVS). Дозволяє зберігати та зчитувати SSID та пароль домашньої мережі як пари "ключ-значення" .
DNSServer.h : Запускається на стандартному UDP порті 53. Реалізує механізм перехоплення запитів, перенаправляючи будь-який введений URL на IP-адресу контролера.
WebServer.h : Реалізує HTTP-сервер на порті 80. Обробляє запити для видачі форми налаштування та прийому нових мережевих параметрів через POST-запит.
WiFi.h : Керує режимами роботи радіомодуля (AP/STA) та процесом підключення до домашньої мережі .
Frontend 
HTML/ CSS: Вебсторінка налаштувань розроблена з використанням Backdrop Filter та адаптивного дизайну для коректного відображення на мобільних пристроях при автоматичному відкритті порталу .

LAB5 - HTTPS

Hardware LAB5:
Мікроконтролер ESP32 DebKit v1: Використовується як обчислювальне ядро, здатне виконувати ресурсомісткі криптографічні операції шифрування та дешифрування в реальному часі.
Вбудована Flash-пам'ять: Використовується для зберігання сертифікатів безпеки X.509 та статичних ресурсів захищеного вебінтерфейсу через файлову систему LittleFS.
Wi-Fi модуль: Забезпечує бездротовий канал зв'язку для передачі зашифрованого TLS-трафіку.
Internal LED: Виступає як об'єкт захищеного керування, доступ до якого надається лише після проходження автентифікаці

ПЗ та бібліотеки LAB5:
OpenSSL (v3.x): Професійний пакет інструментів, використаний на ПК для генерації приватного ключа RSA (2048-bit) та самопідписаного сертифіката сервера (server.crt).
mbedTLS : Криптографічна бібліотека, що забезпечує реалізацію стека TLSv1.2 для шифрування трафіку за стандартом AES-256.Вебсервіси та автентифікація
ESP32_HTTPS_Server: Високорівнева бібліотека для запуску захищеного вебсервера на стандартному порті 443.
Basic Authentication: Механізм перевірки облікових даних користувача через заголовок Authorization. Доступ надається лише за наявності валідного Base64-хендшейку .
FrontendLittleFS : Файлова система для зберігання вебінтерфейсу, доступ до якого обмежений правами авторизації.
ArduinoJson (v7.2.2): Використовується для обміну даними про стан системи в зашифрованому форматі JSON.
Modern JS (Fetch API): Реалізує асинхронні запити до захищеного API контролера, підтримуючи SSL-з'єднання на стороні браузера.
