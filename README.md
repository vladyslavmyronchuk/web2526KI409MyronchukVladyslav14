Програмне забезпечення та бібліотеки - lab1:
HTML / CSS
Використання фреймоврку Arduino для налаштування ESP32
LittleFS : Сучасна файлова система, призначена для мікроконтролерів. Вона використовується для зберігання статичного контенту у Flash-пам'яті пристрою
WiFi.h : Забезпечує роботу стека TCP/IP, дозволяє конфігурувати Wi-Fi модуль для роботи в режимі станції та підключатися до точок доступу.
WebServer.h : Використовується для створення HTTP-сервера, що прослуховує стандартний порт 80. Бібліотека дозволяє налаштовувати маршрути та обробляти вхідні GET-запити від браузера.

Hardware lab2:
Мікроконтролер ESP32 DevKit v1: Виступає в ролі REST-сервера, що обробляє асинхронні запити.
Вбудований світлодіод : Підключений до GPIO2 піна , який виступає об'єктом керування.
Модуль Wi-Fi: Забезпечує бездротовий зв'язок між клієнтом та контролером для передачі HTTP-пакетів.
Flash-пам'ять: Використовується для зберігання файлової системи LittleFS з вебінтерфейсом.


ПЗ та бібліотеки - lab2
Використовується вбудований фреймворк Arduino для налаштування роботи ESP32
LittleFS : Використовується для зберігання файлів Dashboard (HTML/JS). Це дозволяє відокремити інтерфейс користувача від основної логіки керування пінами.
WiFi.h : Забезпечує стабільне підключення до мережі та отримання статичної або динамічної IP-адреси.
WebServer.h : Окрім видачі сторінок на порті 80, бібліотека реалізує API-ендпоїнти:
a)GET /api/status: Повертає поточний стан піна у форматі JSON;
б)POST /api/control: Приймає команди для ввімкнення/вимкнення світлодіода;
ArduinoJson (v7.2.2): Критично важлива бібліотека для десеріалізації вхідних POST-запитів та формування JSON-відповідей про стан системи.
Frontend технології (Fetch)
JavaScript (Fetch API): Використовується для реалізації асинхронної логіки. Замість повного перезавантаження сторінки, браузер у фоновому режимі надсилає запити до ESP32.
setInterval: Функція для автоматичного опитування стану пристрою кожні 2 секунди, що забезпечує актуальність індикатора на екрані.
CSS: Використовується для створення візуального ефекту «лампочки», яка змінює колір залежно від отриманого статусу з API.
